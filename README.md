# DataLoca: Terrestrial Acoustic Localization Data Standard
## Version v0.5

This document describes the **DataLoca** standard for packaging audio clips and metadata that describe sound events and associate spatial position estimates. It defines file layout and column semantics; each dataset also includes a study-specific `readme.md` describing methods and coordinate reference systems (see [template](templates/readme.md)). Typically, this data is generated by deploying audio recorders with precise time-synchronization capabilities (e.g. AudioMoth with on-board GPS), then post-processing audio to detect and estimate spatial positions of sound events.

The key components are (1) a table describing the time and location of the acoustic events (`localized_events.csv`), (2) a set of audio files with subfolders for each recorder (`/audio/`), and (3) metadata providing the spatial positions of each recorder (`/localization_metadata/point_table.csv`).

This standard is maintained in the [DataLoca GitHub repository](https://github.com/LeonardoViotti/data-loca), which will include an index of existing DataLoca-compatible datasets.

[Example dataset on Github](https://github.com/sammlapp/ovenbird_2025_EDI_localization_dataset_labeled_clips/tree/main)

## File structure:

~~~
/project_name/

│	└── environment.yml

	└── /recorder_id001/
	│	└── /synchronized_audio_file_001.wav
	└── /recorder_id002/
	│	└── /synchronized_audio_file_002.wav
	└── ...
			└── ...
~~~


## Terminology:
- **Event**: a sound produced in the world and recorded on multiple microphones
- **Synchronized**: audio files have been post-processed so that they achieve precisely the nominal sample rate and start at precisely the expected time
- **Class**: a sound type category, such as “Northern Cardinal song”. Sound types can refer to species or other categories and should be described in classes.csv or the readme.md file.

Metadata contains a set of tables - as in a “database”, some fields have to key into other tables as exact matches:
- **List columns**: columns that hold multiple values per row (e.g. `file_ids`, `tdoas`) should be encoded as JSON array strings in the CSV (e.g. `["R006.WAV","R002.WAV"]`).
- `point_id` maps between the audio_file_table and point_table
- `file_id`: maps between audio_file_table and localized_events “file_ids” column

#### Column dictionary:
- `event_id`: unique event key. Should be unique for the entire dataset with fixed length and only contain alphanumeric characters and underscores. Examples: `pennsylvania2024grid1_00001`, `event01000`, `0001`
- `label`: species or other sound type name (e.g. “playback”) of the localized event. Should match a value in classes.csv class column
- `start_timestamp`: onset time of the event in ISO format.
- `duration`: event length in seconds. Alternatively, if fixed length clips are shared without event specific bounding boxes it can be the clip length.
- `x`: easting or local x-coordinate (meters) of the localized event estimated position, in the CRS defined in the dataset readme.md
- `y`: northing or local y-coordinate (meters) of the localized event estimated position, in the CRS defined in the dataset readme.md
- [optional] `z`: localized event estimated position altitude in meters. Reference point if sea-level or relative to local position should be specified in readme.md
- `file_ids`: list of file_id for audio clips that participated in the localization. Should match a value in the file_id column of audio_file_table.csv
- `file_start_time_offsets`: time in seconds from start of each audio file to the start of the clip. For example, the event might start 10 seconds into file1.wav and 15 seconds into file2.wav
- [optional] `round`: if multiple rounds of data collection happened for the same array a round identification key, e.g. "2025" and "2024".

#### Other optional Fields:
These are not generated as a single value in correlation-sum approaches, and thus are not required for all datasets
- `tdoas`: list of relative time of arrival in seconds: one per file_id. Arbitrary reference point (often, tdoa=0) is based on the arrival time at one microphone.
- `distance_residuals`: List of residual distance error: one per file_id, in meters
- `classifier_scores`: List of classifier score (float): one per file_id

### [Optional] Classes/sound types description (classes.csv)

This optional file lists and describes all classes (sound types) that occur in the “label” column of localized_events.csv. For instance, “Scarlet Tanager alarm call” might be one class. It can optionally include other columns that assist in the interpretation of the class or detail its taxonomy. If not included, be sure to describe the meaning of the values in “label” column of localized_events.csv in the readme.md file.

## Localization metadata subfolder (localization_metadata/)
These tables join to localized_events.csv via file_id / point_id (see Terminology) 

### Microphone location table (localization_metadata/point_table.csv)
This table organizes the location metadata associated with each point. Point IDs from this table align with Point IDs from the audio file table (audio_file_table.csv). The coordinates refer to the microphone's position in meters, either using a projected CRS (e.g., UTM) or relative position offsets from a point (e.g., center of the array). The coordinate system should be referred to in the dataset readme.md.

| point_id | x | y | z | array_id |
|----------|-------------|---|---|---|----------|
| loca_rail_01  | 61.88444952 | 244.9906262 | 23.176 | A1 |
| loca_rail_02  | 102.7985465 | 229.6864791 | 24.465 | A2 |
| loca_rail_03  | 135.8973872 | 220.6227250 | 24.663 | A2 |


- `point_id`: deployment point key. Should match those in audio_file_table point_id column.
- `x`: easting or local x-coordinate (meters) of the microphone position, in the CRS defined in the dataset readme.md
- `y`: northing or local y-coordinate (meters) of the microphone position, in the CRS defined in the dataset readme.md
- [optional] `z`: microphone altitude in meters. Reference point if sea-level or relative to local position should be specified in readme.md
- [optional] `recorder_id`: recorder key.
- [optional] `array_id` : must include this column if multiple arrays are included in the localization dataset. Each array should have a unique alpha-numeric ID containing only letters, numbers, spaces, and underscores 

- [optional] `accuracy` columns along 3 axes (east-west, north-south, elevation): | accuracy_ew | accuracy_ns | accuracy_elev: These are sometimes provided by RTK GPS. They can be an estimated value if you have not recorded it for each position. readme.md should describe the accuracy metric or how accuracy was measured/approximated
- [optional] `round`: if multiple rounds of data collection happened for the same array with different positions for each recorder, provide a round identification key, e.g. "2025" and "2024". 


### Audio file table (localization_metadata/audio_file_table.csv)
| file_id | relative_path | point_id | start_timestamp |
|---------|---------------|----------|-----------------|
| MSD-0260_20250520_230000.WAV | audio/MSD-0260_20250520_230000.WAV | loca_rail_06 | 2025-05-20T23:00:00.000-05:00 |
| MSD-0260_20250509_120000.WAV | audio/MSD-0260_20250509_120000.WAV | loca_rail_06 | 2025-05-09T12:00:00.000-05:00 |
| MSD-0260_20250509_090000.WAV | audio/MSD-0260_20250509_090000.WAV | loca_rail_06 | 2025-05-09T09:00:00.000-05:00 |

[Example](https://github.com/sammlapp/ovenbird_2025_EDI_localization_dataset_labeled_clips/blob/main/localization_metadata/audio_file_table.csv)

- `file_id`: filename. Should match values in the `file_ids` column of localized_events.csv and the `file_id` column of this table.
- `relative_path`: path to the audio file relative to the top-level of the dataset, e.g. `/audio/recorder001/clip101.wav`.
- `point_id`: deployment point key. Should match `point_id` in point_table.csv.
- `start_timestamp`: recording start time of the audio file in ISO format.
- [optional] `round`: if multiple rounds of data collection happened for the same array a round identification key, e.g. "2025" and "2024".
- [optional] `recorder_id`: recorder key.
- [optional] `card_id`: SD card key used in recorder.


## Audio subfolder (audio/)

This folder contains all audio files referred to in the localized_events.csv table. Audio files might be synchronized short clips extracted around localized events, or longer files containing the localized events within them. Organization of files within /audio/ folder is flexible including sub-folders as long as that is reflected on associated metadata sheets `relative_path` columns. Each audio file will have a unique `file_id` and a row in localization_metadata/audio_file_table.csv (the table contains the `file_id`, its relative path eg /audio/subfolder/file1.wav, and the point_id where it was recorded). 


### Example 1: Short audio clips in a flat structure
```
audio/
├── clip0001.flac
├── clip0002.flac
└── ...
```
localization_metadata/audio_file_table.csv first row:

| file_id | relative_path | point_id | start_timestamp |
| --- | --- | --- | --- | 
| clip0001 | audio/clip0001.flac | ABC12 | 2025-01-01T06:00:00.000-05:00 |

## Scripts subfolder (scripts/)

**All scripts used to produce the localized events (.py, .r, .rmd, .ipynb)**  
Code scripts and notebooks (R, python,  etc) used to generate detections and localizations  
- If there is no hand-labeling or manual review of clips: should be able to re-create localized_events.csv from these scripts
- If there is a hand-labeling/manual review step: include any scripts/workflows used to review events, e.g. if you used an interactive notebook, if you produced a table of events to exclude/include, etc.

**Environment file(s)**  
Frozen Python environment files (.yml) listing package versions
- If using the anaconda environment manager you can create this file with `conda env export -f myenv.yml`

## [Optional] Observed events subfolder (observed_events/)

Subfolder containing records of events seen (or produced) in person with position location or estimates (accuracy of position estimates should go in readme.md). All of these are optional.

### Acoustic playback experiments (observed_events/playbacks.csv)

| playback_id | label | start_timestamp | duration | x | y | z | file_id | file_start_time_offsets |
| --- | --- | --- | --- | --- | --- | --- | --- | --- |
| playback001 | KAAM_playback | 2025-07-20T10:00:00.000-05:00 | 100 | 662516.184452 | 6124222.74555885 | nan | | |

Columns:
- `playback_id`: unique within the dataset; fixed length; only alphanumeric characters and underscores
- `label`: species or other class name (e.g. “tone sweep”, “species mix”) of the playback, included in classes.csv
- `start_timestamp`: onset time of playback in ISO format 2025-05-20T10:00:00.000-05:00. Note that these are event specific, not specific for each file.
- `duration`: playback length in seconds
- `x`: easting or local x-coordinate (meters) of the event position, in the CRS defined in the dataset readme.md
- `y`: northing or local y-coordinate (meters) of the event position, in the CRS defined in the dataset readme.md
- [optional] `z`: event altitude in meters. Reference point if sea-level or relative to local position should be specified in readme.md
- `file_id`
- `file_start_time_offsets`
- [optional] `notes`

### Field observations with positions (observed_events/observations.csv)

Table of human field observations of acoustic events with known positions
| observed_event_id | label | start_timestamp | duration | x | y | z | direction |
| --- | --- | --- | --- | --- | --- | --- | --- |
| focalfollow01 | KAAM_song_a | 2025-05-20T10:45:00 | 5 | 662516.184452 | 6124222.74555885 | nan | 15 degrees |
| focalfollow02 | KAAM_song_b | 2025-05-20T11:06:00 | 45 | 662516.184452 | 6124222.74555885 | nan | 60 degrees |

Columns:
- `observed_event_id`: unique within the dataset; fixed length; only alphanumeric characters and underscores
- `label`: species or other class name (e.g. “songsparrow_aggression”) of the observed event
- `start_timestamp`: onset time of the observation in ISO format 2025-05-20T10:00:00.000-05:00
- `duration`: event length in seconds
- `x`: easting or local x-coordinate (meters) of the event position, in the CRS defined in the dataset readme.md
- `y`: northing or local y-coordinate (meters) of the event position, in the CRS defined in the dataset readme.md
- [optional] `z`: event altitude in meters. Reference point if sea-level or relative to local position should be specified in readme.md
- [optional]: direction (degrees CW from N): orientation of animal when sound was produced
- [optional] `notes`

Can include files for other observation types, such as:  
- Spot mapping  
- Point count survey  
- Transect survey  
- Focal follows
